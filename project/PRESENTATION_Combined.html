Approach Result

Conclusion

  • Data problem: Too few examples in 40+ hour range

Feature Engineering Deep Dive

User Average Energy: Capturing Behavior Patterns

The Idea

Each user has consistent charging habits. Some always charge 10-15 kWh. Others charge 30-40 kWh. By capturing this pattern with aggregate data, we help the neural network predict session outcomes.

The Formula

user_avg_energy = mean(energy_session) for all sessions by that user

Example Calculation

User Session 1 (kWh) Session 2 (kWh) Session 3 (kWh) Average (Feature)
User A 12.5 11.8 13.2 12.5 kWh
User B 35.2 36.1 34.9 35.4 kWh
User C 22.3 28.5 25.1 25.3 kWh

Implementation in Python

import pandas as pd

# Calculate average energy per user
user_avg_energy = df.groupby('user_id')['energy_session'].mean()

# Map back to all sessions
df['user_avg_energy'] = df['user_id'].map(user_avg_energy)

# Result: Each session now has its user's average energy
print(df[['user_id', 'energy_session', 'user_avg_energy']].head())
#   user_id  energy_session  user_avg_energy
# 0   user_1           15.2             12.5
# 1   user_1           10.1             12.5
# 2   user_2           34.8             35.4
# 3   user_3           26.5             25.3

Why This Helps Neural Networks

Domain Insight: Users with larger average energy tend to have longer sessions and park longer.

Network Benefit: Instead of learning user patterns from scratch, the network gets a direct clue about that user's typical behavior.

Impact: This single feature improves prediction accuracy by ~5% because it reduces noise and captures user-level patterns.

Three Key Insights

1. Data Quality > Model Complexity

  • Simple tree on good data > complex NN on bad data
  • 328% improvement from data, not algorithms

2. Problem Framing Matters

  • Regression failed: R² = 0.59
  • Classification succeeded: AUC = 0.847 (same data)
  • Right question > Fancy model

3. Know Your Tools

  • NNs: 30K+ samples. Trees: 6K samples.
  • Trees for tabular. NNs for images/sequences.
  • Match tool to data, not vice versa
